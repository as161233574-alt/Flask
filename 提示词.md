## 详细需求

### 数据模型
User 表：id, username(唯一), email(唯一), password_hash, role('user'|'admin'),
         created_at, updated_at
         方法：set_password(plain), check_password(plain), to_dict()
TokenBlocklist 表：id, jti(唯一), created_at

### API 端点
POST   /auth/register          无需认证，body: {username, email, password}，密码最少6位
POST   /auth/login             无需认证，body: {email, password}，返回 {access_token, refresh_token}
POST   /auth/logout            Bearer token，将 jti 写入 TokenBlocklist
POST   /auth/refresh           Refresh token，返回新 access_token
GET    /users/me               Bearer token，返回当前用户信息
PUT    /users/me               Bearer token，body: {username?, email?}，更新个人信息
PUT    /users/me/password      Bearer token，body: {current_password, new_password}
GET    /users/                 Bearer token + role==admin，返回所有用户列表
PUT    /users/<int:id>/role    Bearer token + role==admin，body: {role}，修改用户角色

### 统一响应格式
成功：{"success": true, "data": {...}}
失败：{"success": false, "error": "错误描述"}
HTTP 状态码：200 成功，201 创建，400 参数错误，401 未认证，403 无权限，404 未找到，409 冲突

### 配置类
DevelopmentConfig：DEBUG=True，SQLite 文件
TestingConfig：TESTING=True，sqlite:///:memory:，JWT_ACCESS_TOKEN_EXPIRES=False，JWT_REFRESH_TOKEN_EXPIRES=False
ProductionConfig：从环境变量读取

### JWT 黑名单
在 app/__init__.py 中注册 @jwt.token_in_blocklist_loader，
查询 TokenBlocklist 表判断 jti 是否已被撤销。
logout 时从 get_jwt()["jti"] 获取 jti 并插入 TokenBlocklist。

### admin_required 装饰器
位于 app/utils/decorators.py，使用 @wraps(fn) + @jwt_required()，
通过 get_jwt_identity() 查询用户，role != 'admin' 时返回 403 JSON。

### 测试要求（必须达到 100% 覆盖率）
conftest.py 提供以下 fixture：
  - app：使用 TestingConfig 创建应用
  - client：scope=function，每次测试前 create_all，结束后 drop_all
  - user_token：注册普通用户并返回 access token 字符串
  - user_refresh_token：注册普通用户并返回 refresh token 字符串
  - admin_token：注册用户后将 role 改为 admin，返回 access token 字符串

test_auth.py 覆盖所有分支：
  - register：成功、缺少字段、密码过短、用户名重复、邮箱重复
  - login：成功、缺少字段、密码错误、用户不存在
  - logout：成功、已撤销 token 被拒绝、无 token
  - refresh：成功、无 token

test_user.py 覆盖所有分支：
  - GET /users/me：成功、无 token
  - PUT /users/me：更新用户名成功、更新邮箱成功、空字段、用户名冲突、邮箱冲突
  - PUT /users/me/password：成功、缺少字段、当前密码错误、新密码过短
  - GET /users/：admin 成功、普通用户 403、无 token 401
  - PUT /users/<id>/role：admin 成功、无效 role、用户不存在、普通用户 403、无 token 401

### Docker 部署
Dockerfile：

  WORKDIR /app
  COPY requirements.txt .
  RUN pip install --no-cache-dir -r requirements.txt
  COPY . .
  EXPOSE 8000
  CMD ["gunicorn", "-w", "4", "-b", "0.0.0.0:8000", "run:app"]

docker-compose.yml：
  服务名 web，env_file .env，端口 8000:8000，
  挂载命名卷 sqlite_data:/app/data

      app.run()
